# CWE-1241 - Use of Predictable Algorithm in Random Number Generator

## CWE Explanation

This weakness occurs when a device relies on a Random Number Generator (RNG) that uses a predictable algorithm, such as a Pseudo-Random Number Generator (PRNG) or a Linear Feedback Shift Register (LFSR), for security-critical operations. Unlike True Random Number Generators (TRNGs) which derive entropy from unpredictable physical phenomena (like thermal noise), predictable algorithms are deterministic. If an attacker can determine the initial seed or the current internal state of the generator, they can predict all future outputs.

In hardware designs, developers might choose LFSRs for their simplicity and low resource usage. However, LFSRs are mathematically linear and easily reversible. Even more complex PRNGs, if not specifically designed for cryptography (CSPRNGs), can exhibit biases or have cycles short enough to be exploited. When these are used to generate keys, nonces, or initialization vectors, the entire security of the cryptographic system collapses.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | High (Keys can be predicted) |
| **Integrity** | High (Nonces for signatures can be predicted) |

## Attack Scenario

An attacker targets a System-on-Chip (SoC) that uses a Linear Feedback Shift Register (LFSR) to generate random "challenges" for an authentication protocol. In the next step, the attacker initiates multiple authentication attempts and records the challenges sent by the SoC. Because the LFSR's output is deterministic and based on a linear recurrence relation, the attacker uses the collected samples to solve for the LFSR's feedback polynomial and current state. Once the state is known, the attacker can perfectly predict the next challenge the SoC will generate. This allows the attacker to pre-calculate the correct response (if the key is known or if the protocol allows for replay/pre-computation attacks based on known challenges), bypassing the authentication mechanism.

## MITRE Reference Data

### Description

The device uses an algorithm that is predictable and generates a pseudo-random number.

### Extended Description

Pseudo-random number generator algorithms are predictable because their registers have a finite number of possible states, which eventually lead to repeating patterns. As a result, pseudo-random number generators (PRNGs) can compromise their randomness or expose their internal state to various attacks, such as reverse engineering or tampering.

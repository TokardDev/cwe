# CWE-1351 - Improper Handling of Hardware Behavior in Exceptionally Cold Environments
## CWE Explanation

This weakness arises when a hardware security primitive, such as a Physical Unclonable Function (PUF) or a Random Number Generator (RNG), relies on physical properties of memory (like SRAM start-up values) that change behavior under extreme cold. Typically, volatile memory like SRAM loses its data and settles into a random, manufacturing-defined pattern when powered up. However, at exceptionally low temperatures, the phenomenon of data remanence increases significantly. Data from a previous power cycle may persist even after power is removed and restored.

If a security mechanism assumes that a power cycle always results in a fresh, high-entropy state (based on manufacturing variations), but instead reads stale data from the previous session due to the cold environment, the "unclonable" or "random" value becomes predictable or controllable by an attacker who can manipulate the previous state.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | High (PUF keys or RNG seeds become predictable) |
| **Integrity** | High (Authentication relying on PUF can be spoofed) |
| **Availability** | Low |

## Attack Scenario

An attacker targets an IoT device that uses an SRAM-based Physical Unclonable Function (PUF) to generate a unique device root key upon boot. The PUF logic assumes that when the device is powered on, the SRAM cells will settle into a random state determined by their physical manufacturing characteristics. The attacker cools the device to an extremely low temperature (e.g., using freeze spray or liquid nitrogen) and powers it on. They then load a known pattern into the SRAM memory area used by the PUF. Without letting the device warm up, the attacker quickly power-cycles the device. Due to the low temperature, the SRAM cells retain the attacker-controlled pattern instead of settling into their natural preferred state. The PUF generation logic then processes this known pattern to derive the root key. The attacker effectively forces the device to generate a known, insecure key instead of the unique secret key, allowing them to bypass device authentication or decrypt stored data.

## MITRE Reference Data

### Description

A hardware device, or the firmware running on it, is missing or has incorrect protection features to maintain goals of security primitives when the device is cooled below standard operating temperatures.

### Extended Description

The hardware designer may improperly anticipate hardware behavior when exposed to exceptionally cold conditions. As a result they may introduce a weakness by not accounting for the modified behavior of critical components when in extreme environments.

An example of a change in behavior is that power loss won't clear/reset any volatile state when cooled below standard operating temperatures. This may result in a weakness when the starting state of the volatile memory is being relied upon for a security decision. For example, a Physical Unclonable Function (PUF) may be supplied as a security primitive to improve confidentiality, authenticity, and integrity guarantees. However, when the PUF is paired with DRAM, SRAM, or another temperature sensitive entropy source, the system designer may introduce weakness by failing to account for the chosen entropy source's behavior at exceptionally low temperatures. In the case of DRAM and SRAM, when power is cycled at low temperatures, the device will not contain the bitwise biasing caused by inconsistencies in manufacturing and will instead contain the data from previous boot. Should the PUF primitive be used in a cryptographic construction which does not account for full adversary control of PUF seed data, weakness would arise.

This weakness does not cover "Cold Boot Attacks" wherein RAM or other external storage is super cooled and read externally by an attacker.

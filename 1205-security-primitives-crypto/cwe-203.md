## CWE Explanation

This weakness occurs when a product exposes differences in its behavior or response—such as error messages, timing, or resource consumption—based on the internal state or the sensitivity of the data being processed. These observable discrepancies create side channels that allow unauthorized actors to infer sensitive information, such as validity of credentials, cryptographic keys, or internal logic paths.

In a security context, particularly within cryptographic primitives or authentication mechanisms, operations should ideally be constant-time and constant-resource, or at least indistinguishable regardless of the input secrets. When a system fails to maintain this uniformity, an attacker can meticulously measure these discrepancies (e.g., measuring the time it takes to reject a password or observing cache evictions) to deduce the secret data bit by bit.

The root cause often lies in optimizations that prioritize performance over security (like early exit in comparison loops) or conditional logic that handles valid and invalid inputs differently, inadvertently signaling the internal state to the outside world.

## Attack Scenarios

### Scenario 1
An attacker is able to interact with an authentication interface. In the next step, the attacker attempts to log in with various usernames. The system returns "User not found" for invalid usernames and "Incorrect password" for valid usernames. By observing these different error messages, the attacker successfully enumerates valid accounts on the system, which can then be targeted for password guessing attacks.

### Scenario 2
An attacker is able to measure the execution time of a password validation process or cryptographic operation. The system uses a comparison function that exits immediately upon finding the first mismatched character or byte. In the next step, the attacker sends multiple inputs and measures the response time with high precision. Inputs that match more of the secret (e.g., the first few characters of a password) take slightly longer to process than those that fail immediately. By statistically analyzing these timing differences, the attacker can recover the secret password or key character by character.

### Scenario 3
An attacker is able to run code on the same physical hardware as a victim process, sharing microarchitectural resources like the processor cache. The victim executes an encryption routine where the memory access pattern depends on the value of the secret key (e.g., table lookups). In the next step, the attacker primes the cache and then allows the victim to execute. After the victim's execution, the attacker measures the time to access their own memory lines. Cache lines that were evicted by the victim's operation reveal which memory locations the victim accessed. This discrepancy in cache state allows the attacker to deduce information about the secret key used by the encryption routine.

## MITRE Reference Data

### Description

The product behaves differently or sends different responses under different circumstances in a way that is observable to an unauthorized actor, which exposes security-relevant information about the state of the product, such as whether a particular operation was successful or not.

### Extended Description

Discrepancies can take many forms, and variations may be detectable in timing, control flow, communications such as replies or requests, or general behavior. These discrepancies can reveal information about the product's operation or internal state to an unauthorized actor. In some cases, discrepancies can be used by attackers to form a side channel.

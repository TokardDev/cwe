## CWE Explanation

This weakness occurs when a hardware device's physical characteristics—specifically its power consumption, electromagnetic emissions (EME), or even sound—leak information about the data it is processing. In digital logic, switching a transistor from 0 to 1 consumes a tiny amount of power and generates a faint electromagnetic field. When operations on sensitive data (like encryption keys or PINs) result in distinct patterns of switching activity, an attacker with physical access and sensitive measurement equipment (like an oscilloscope or EM probe) can correlate these physical fluctuations with the internal secrets.

For instance, if a cryptographic algorithm processes a '1' bit in a key differently than a '0' bit (e.g., performing an extra multiplication step for a '1'), the power consumption trace will show a distinct spike or longer duration. By collecting and analyzing these traces (Simple Power Analysis - SPA, or Differential Power Analysis - DPA), an attacker can extract the entire secret key without ever breaking the mathematical algorithm itself.

## Attack Scenarios

### Scenario 1
An attacker has physical access to a hardware security module (HSM) that performs RSA decryption. The RSA implementation uses a standard "square-and-multiply" algorithm for modular exponentiation. In this algorithm, the hardware performs a multiplication operation only if the current bit of the private exponent is '1', but skips it if the bit is '0'. The attacker attaches a high-precision resistor in series with the device's power supply and connects an oscilloscope to measure the voltage drop (which is proportional to power consumption). By observing the power trace during a decryption operation, the attacker can clearly distinguish the time periods where the extra multiplication occurs (high power consumption) versus where it is skipped. This visual pattern allows the attacker to directly read the bits of the private key (Simple Power Analysis).

### Scenario 2
An attacker targets a secure USB token used for two-factor authentication. The token uses a microcontroller to sign challenges using an internal private key. The attacker places a sensitive electromagnetic (EM) probe near the microcontroller package while it is performing the signing operation. The attacker captures thousands of EM traces corresponding to different known challenges. Because the microcontroller leaks information about the intermediate values of the cryptographic calculation through its electromagnetic emissions, the attacker employs Differential Electromagnetic Analysis (DEMA). By statistically correlating the variations in the EM field with hypothetical values of the key bits, the attacker successfully extracts the private key from the token.

## MITRE Reference Data

### Description

The device does not contain sufficient protection mechanisms to prevent physical side channels from exposing sensitive information due to patterns in physically observable phenomena such as variations in power consumption, electromagnetic emissions (EME), or acoustic emissions.

### Extended Description

An adversary could monitor and measure physical phenomena to detect patterns and make inferences, even if it is not possible to extract the information in the digital domain.

Physical side channels have been well-studied for decades in the context of breaking implementations of cryptographic algorithms or other attacks against security features. These side channels may be easily observed by an adversary with physical access to the device, or using a tool that is in close proximity. If the adversary can monitor hardware operation and correlate its data processing with power, EME, and acoustic measurements, the adversary might be able to recover of secret keys and data.

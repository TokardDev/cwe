# CWE-1263 - Improper Physical Access Control

## CWE Explanation

This weakness exists when a product fails to implement sufficient physical protection mechanisms to restrict access to sensitive components or information. Hardware devices often operate in uncontrolled environments where physical access by an attacker is a realistic threat. If the device does not employ physical security measures—such as tamper-evident seals, hardened enclosures, potting materials, or active tamper detection circuits—an attacker can directly interact with the internal hardware.

Improper physical access control allows attackers to bypass logical security boundaries. They might probe test points, read from debug interfaces (JTAG/UART) that should be inaccessible, directly tap bus lines to intercept data, or physically modify components (e.g., fault injection, replacing chips).

The root cause is typically a design decision that prioritizes cost or aesthetics over physical security, or an assumption that the device will only operate in a physically secure environment.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can read sensitive data by physically probing memory or buses |
| **Integrity** | The attacker can modify firmware or data by replacing components or injecting signals |
| **Availability** | The attacker can physically damage or disable the device |

## Attack Scenario

An attacker gains physical possession of an IoT edge device deployed in a public location. The device casing is secured only by standard screws and lacks any intrusion detection sensors or tamper switches. The attacker opens the case without triggering any alarms or data zeroization mechanisms. Once inside, they attach logic analyzer probes to the exposed SPI bus connecting the main processor to the flash memory. By capturing the traffic during a boot sequence, the attacker extracts the unencrypted firmware and sensitive configuration secrets, compromising the device and potentially the wider network.

## MITRE Reference Data

### Description

The product is designed with access restricted to certain information, but it does not sufficiently protect against an unauthorized actor with physical access to these areas.

### Extended Description

Sections of a product intended to have restricted access may be inadvertently or intentionally rendered accessible when the implemented physical protections are insufficient. The specific requirements around how robust the design of the physical protection mechanism needs to be depends on the type of product being protected. Selecting the correct physical protection mechanism and properly enforcing it through implementation and manufacturing are critical to the overall physical security of the product.

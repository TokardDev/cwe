# CWE-1422 - Exposure of Sensitive Information caused by Incorrect Data Forwarding during Transient Execution

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker is able to read memory |
| **Integrity** | Low impact |
| **Availability** | Low impact |

## Attack Scenarios

### Scenario 1
An attacker is able to identify a vulnerable code sequence (gadget) within a victim process or a high-trust application. In the next step, the attacker triggers a processor event, such as a Page Fault, while the processor is handling a data store operation. Due to the specific hardware design, the processor incorrectly forwards data from an internal buffer (like the store buffer) to a subsequent transient operation before the fault is resolved. This forwarded data is often stale or incorrect, but the transient instruction uses it to perform a memory access dependent on this value. Consequently, the attacker can analyze the cache state via a side-channel to recover the sensitive data that was transiently accessed using the incorrect forwarding.

### Scenario 2
An attacker is able to run code within a restricted sandbox. Since some processors try to predict when a store will forward data to a subsequent load, the attacker trains the CPU's memory predictor to believe that a specific load instruction depends on a previous store instruction. In the next step, the attacker executes the code in a state where the dependency does not exist, but the predictor still speculates that it does. The CPU transiently forwards stale data to the load instruction, effectively bypassing software bounds checking. The attacker uses this transient window to leak data from outside the sandbox by encoding the stale value into the cache hierarchy, which is retrieved via timing analysis.

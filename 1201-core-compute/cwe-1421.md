# CWE-1421 - Exposure of Sensitive Information in Shared Microarchitectural Structures during Transient Execution

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker is able to read memory |
| **Integrity** | Low impact |
| **Availability** | Low impact |

## Attack Scenarios

### Scenario 1
An attacker is able to corrupt a low trust application and attempts to read a memory address restricted to the kernel. In this scenario, the processor performs the access control check (verifying if the address is mapped to user or kernel space) in parallel with the memory read operation. Consequently, the processor transiently returns the restricted kernel data from the shared cache before the permission check officially fails. The attacker utilizes a gadget to transmit this result over a covert channel before the pipeline is flushed, bypassing the memory isolation.

### Scenario 2
An attacker is able to execute unprivileged code and attempts to read a system register restricted to a higher privilege level. In a normal case, the hardware prevents this access and would raise an exception. However, the attacker manipulates the branch prediction system (e.g. via a conditional branch CBZ [Compare and Branch on Zero instruction]) to mispredict a path where the read instruction is executed transiently. During this transient window, the processor reads the value of the restricted register before the misprediction is resolved. The attacker then uses a subsequent memory access instruction to encode this restricted value into a cache covert channel, making it retrievable even after the pipeline is flushed.

### Scenario 3
An attacker runs on a hardware thread that shares physical resources with a victim thread on the same processor core. The victim's data passes through shared microarchitectural structures known as fill buffers (used for temporary storage between caches and memory). The attacker triggers a transient execution event, specifically via a faulting operation or a microcode assist. This event allows the attacker to transiently read data currently residing in the shared fill buffer, effectively bypassing the logical isolation between the two threads. The attacker extracts this data using a side-channel analysis before the processor clears the buffer state.

# CWE-1342 - Information Exposure through Microarchitectural State after Transient Execution


## Related CVEs

| CVE | Description |
|-----|-------------|
| [CVE-2020-0551](https://www.cve.org/CVERecord?id=CVE-2020-0551) | Load value injection in some processors utilizing speculative execution may allow an authenticated user to enable information disclosure via a side-channel with local access. |

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can extract confidential data |
| **Integrity** | The attacker can execute commands and modify memory |
| **Availability** | Low impact (An availability loss may be a result of integrity loss, but not a direct consequence of the CWE) |

## Attack Scenario

An attacker is able to corrupt a low trust application from a low trust interface and identify a vulnerable code (gadget) in a high trust application. First, the attacker triggers transient execution (via a Page Fault, a General Protection Fault, a manipulation of Branch Prediction system or Microcode Assists for e.g.) to force the CPU to use incorrectly predicted data and alter a microarchitectural buffer. After a flush, the vulnerability described by this CWE prevents the cache from properly cleaning up, allowing the attacker to extract it through a side-channel analysis.

## Remediations from MITRE

| Phase | Strategy | Mitigation | Effectiveness |
|-------|----------|------------|---------------|
| Architecture and Design; Requirements | - | Hardware ensures that no illegal data flows from faulting micro-ops exists at the microarchitectural level. Being implemented in silicon it is expected to fully address the known weaknesses with limited performance impact. | High |
| Build and Compilation | - | Include instructions that explicitly remove traces of unneeded computations from software interactions with microarchitectural elements e.g. lfence, sfence, mfence, clflush. This effectively forces the processor to complete each memory access before moving on to the next operation. This may have a large performance impact. | High |

*Source: MITRE CWE Database*

## Recommended Remediations

| Phase | Remediation | Priority |
|-------|-------------|----------|
| Architecture and Design | Design the microarchitecture to ensure that transient execution results in faulting micro-ops are squashed and their microarchitectural side effects (cache state, TLB state, buffer contents) are fully rolled back before any subsequent instruction can observe them | Critical |
| Architecture and Design | Partition microarchitectural buffers (line fill buffers, load ports, store buffers) between security domains to prevent cross-domain information leakage via transient execution | Critical |
| Build and Compilation | Insert serialization barriers (LFENCE or equivalent) after security-critical bounds checks and before memory accesses that could be speculatively executed with incorrect data | High |
| Testing | Validate microarchitectural state cleanup using side-channel analysis test suites that attempt to recover data from transient execution across privilege boundaries | High |
| Documentation | Document all known transient execution behaviors and recommended software mitigations in the processor's security reference manual | Medium |

*These remediations are recommended best practices and are not sourced from MITRE.*

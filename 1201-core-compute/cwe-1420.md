# CWE-1420 - Exposure of Sensitive Information during Transient Execution


## Related CVEs

| CVE | Description |
|-----|-------------|
| [CVE-2017-5753](https://www.cve.org/CVERecord?id=CVE-2017-5753) | Microarchitectural conditional branch predictors may allow operations to execute transiently after a misprediction, potentially exposing data over a covert channel. |
| [CVE-2021-0089](https://www.cve.org/CVERecord?id=CVE-2021-0089) | A machine clear triggered by self-modifying code may allow incorrect operations to execute transiently, potentially exposing data over a covert channel. |
| [CVE-2022-0002](https://www.cve.org/CVERecord?id=CVE-2022-0002) | Microarchitectural indirect branch predictors may allow incorrect operations to execute transiently after a misprediction, potentially exposing data over a covert channel. |

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can extract confidential data and read memory |
| **Integrity** | Low impact |
| **Availability** | Low impact |

## Attack Scenarios

### Scenario 1
An attacker is able to corrupt a low trust application from a low trust interface. In the next step, the attacker is able to execute malicious code on the CPU. Via this code, the attacker is able to trigger transient execution and use the transient execution weakness to expose data via a covert channel (using timing analysis for e.g.) that is accessible within an attacker's processor context.

### Scenario 2
An attacker is able to identify a vulnerable code sequence in a high trust application. After this, the attacker uses the vulnerable code (gadget) while triggering the transient execution (via a Page Fault, a General Protection Fault, a manipulation of Branch Prediction system or Microcode Assists for e.g.) to perform a transient execution to expose data via a covert channel (using timing analysis for e.g.) that is accessible within the victim's processor context, via Out of Bounds (forces the CPU to check a data outside of the normal value expected by the program, to expose a close data in term of memory address) for example.

# CWE-1252 - CPU Hardware Not Configured to Support Exclusivity of Write and Execute Operations

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can execute unauthorized code and commands |
| **Integrity** | The attacker can execute unauthorized code and commands |
| **Availability** | Low impact (An availability loss may be a result of integrity loss, but not a direct consequence of the CWE) |

## Attack Scenario

An attacker is able to corrupt a low trust application via a low trust interface and load a malicious program into memory. In a normal case, and more generally in the majority of recent hardware, systems use a special bit to distinguish between memory types (instructions, which can be executed, versus data, which should not). However, in this scenario, the CPU lacks an MMU/MPU (Memory Management Unit / Memory Protection Unit). This lack of hardware prevents the CPU from distinguishing if the memory content is an instruction or data, as it lacks the hardware gate to refuse execution. Consequently, the attacker can trigger the execution of the injected data, leading to the execution of malicious code and allowing the attacker to gain full control over the system.

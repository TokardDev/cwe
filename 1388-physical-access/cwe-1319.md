# CWE-1319 - Improper Protection against Electromagnetic Fault Injection (EM-FI)

## CWE Explanation

This weakness exists when a hardware device lacks sufficient shielding, sensors, or design redundancy to protect against Electromagnetic Fault Injection (EM-FI). In an EM-FI attack, an adversary generates a focused, high-intensity magnetic pulse near the surface of the integrated circuit. This pulse induces transient currents in the chip's internal interconnects via electromagnetic induction.

These induced currents can cause momentary voltage glitches that disrupt the device's normal operation. Common effects include flipping bits in memory or registers, corrupting data in transit on internal buses, or causing the processor to misinterpret or skip instructions. By carefully timing these pulses, attackers can bypass critical security checks, disable protection features, or force the device into an undefined state where secrets can be extracted.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can bypass readout protection or password checks to access sensitive firmware or data |
| **Integrity** | The attacker can modify the contents of registers, memory, or the instruction stream |
| **Availability** | The attack may cause the device to crash or reset, though this is often a side effect rather than the goal |
| **Access Control** | Secure Boot, JTAG locks, and authentication mechanisms can be completely bypassed |

## Attack Scenarios

### Scenario 1
An attacker targets a System-on-Chip (SoC) during its power-on reset sequence. The device is designed to read immutable security configuration bits (fuses) and copy them into volatile shadow registers that control runtime security settings (such as "Secure Boot Enabled" or "JTAG Locked"). The attacker positions an EM probe over the chip and triggers a pulse exactly when this transfer occurs. The induced glitch corrupts the value on the internal data bus, causing the shadow registers to latch a "disabled" or "unlocked" state. The device proceeds to boot with its security mechanisms effectively turned off.

### Scenario 2
An attacker attempts to gain access to a debug interface on a microcontroller that is protected by a password check. By analyzing the device's power consumption traces (Side-Channel Analysis), the attacker identifies the precise clock cycle where the processor compares the entered password against the stored correct value. The attacker then injects a targeted electromagnetic fault at that exact moment. The glitch causes the processor to skip the conditional branch instruction that would normally handle a failed check, allowing the attacker to proceed as if the correct password had been entered.

## MITRE Reference Data

### Description

The device is susceptible to electromagnetic fault injection attacks, causing device internal information to be compromised or security mechanisms to be bypassed.

### Extended Description

Electromagnetic fault injection may allow an attacker to locally and dynamically modify the signals (both internal and external) of an integrated circuit. EM-FI attacks consist of producing a local, transient magnetic field near the device, inducing current in the device wires. A typical EMFI setup is made up of a pulse injection circuit that generates a high current transient in an EMI coil, producing an abrupt magnetic pulse which couples to the target producing faults in the device, which can lead to:
* Bypassing security mechanisms such as secure JTAG or Secure Boot
* Leaking device information
* Modifying program flow
* Perturbing secure hardware modules (e.g. random number generators)

### Demonstrative Examples

#### Example 1
In many devices, security related information is stored in fuses. These fuses are loaded into shadow registers at boot time. Disturbing this transfer phase with EM-FI can lead to the shadow registers storing erroneous values potentially resulting in reduced security.

Colin O'Flynn has demonstrated an attack scenario [REF-1144] that uses electro-magnetic glitching during booting to bypass security and gain read access to flash, read and erase access to shadow memory area (where the private password is stored). Most devices in the MPC55xx and MPC56xx series that include the Boot Assist Module (BAM) (a serial or CAN bootloader mode) are susceptible to this attack. In this paper, a GM ECU was used as a real life target. While the success rate appears low (less than 2 percent), in practice a success can be found within 1-5 minutes once the EMFI tool is setup. In a practical scenario, the author showed that success can be achieved within 30-60 minutes from a cold start.

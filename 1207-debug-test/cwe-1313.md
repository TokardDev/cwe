# CWE-1313 - Hardware Allows Activation of Test or Debug Logic at Runtime

## CWE Explanation

This weakness occurs when a hardware product allows the activation of powerful test or debug logic during normal system runtime. Test and debug features (such as error injection, scan chain access, or internal state dumping) are designed for manufacturing and development phases. If these features remain accessible or can be re-enabled while the device is in its production/runtime state, they can be abused by an attacker to alter the system's behavior or leak sensitive data.

An attacker could use error injection to corrupt data on a bus, leading to unintended command execution or denial of service. They could also use runtime debug access to read secret keys or bypass security checks. In a secure design, these features should be permanently disabled (e.g., via fuse blowing) or strictly locked behind a strong authentication mechanism that is only active during a secure boot window.

The root cause is a failure to enforce state-based restrictions on debug and test logic, allowing highly privileged hardware operations to be triggered from unprivileged or normal operating contexts.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | High - Attacker can read sensitive data using runtime debug logic. |
| **Integrity** | High - Attacker can use error injection or debug overrides to modify system behavior. |
| **Availability** | High - Attacker can halt the processor or cause system instability. |

## Attack Scenario

An attacker discovers a hardware test feature that allows for manual error injection into the internal processor pipeline, intended for verifying error-correction logic. In the next step, the attacker triggers this feature at runtime while the processor is performing a critical security check, such as an identity verification for a high-privilege operation. By carefully timing the error injection to flip a bit in the status register, the attacker successfully bypasses the authentication check and gains administrative access to the system.

## MITRE Reference Data

### Description

During runtime, the hardware allows for test or debug logic (feature) to be activated, which allows for changing the state of the hardware. This feature can alter the intended behavior of the system and allow for alteration and leakage of sensitive data by an adversary.

### Extended Description

An adversary can take advantage of test or debug logic that is made accessible through the hardware during normal operation to modify the intended behavior of the system. For example, an accessible Test/debug mode may allow read/write access to any system data. Using error injection (a common test/debug feature) during a transmit/receive operation on a bus, data may be modified to produce an unintended message. Similarly, confidentiality could be compromised by such features allowing access to secrets.

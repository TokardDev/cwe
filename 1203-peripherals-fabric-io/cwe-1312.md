# CWE-1312 - Missing Protection for Mirrored Regions in On-Chip Fabric Firewall

## CWE Explanation

This weakness occurs in System-on-Chip (SoC) architectures where memory ranges are mirrored for redundancy or fault tolerance, but the on-chip fabric firewall fails to extend its protection to these mirrored regions. Address mirroring maps the same physical memory to multiple address ranges. If a security policy restricts access to the primary address range (the "original"), it is critical that the same restrictions apply to all mirrored copies of that range.

If the firewall or access control unit only checks transactions against the primary address range, an attacker can exploit this oversight. By targeting the mirrored address instead of the protected primary address, the attacker can access the same underlying physical data.

This effectively bypasses the intended access control mechanism. The root cause is typically an incomplete firewall configuration or a design flaw where the relationship between original and mirrored regions is not fully accounted for in the security logic.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can read sensitive data from the mirrored region, bypassing read protections on the primary region. |
| **Integrity** | The attacker can modify data in the mirrored region, corrupting the protected memory content. |
| **Availability** | Low impact (*System instability is possible if critical configuration data is corrupted via the mirror*). |

## Attack Scenario

An attacker running unprivileged code wishes to modify a protected configuration register located at physical address `0x1000`. The system firewall correctly blocks unprivileged write attempts to the range `0x1000 - 0x1FFF`. However, the memory controller mirrors this range to `0x5000 - 0x5FFF` for legacy support or redundancy, and the firewall is not configured to protect this secondary range. The attacker issues a write transaction to `0x5000` (which aliases to `0x1000`). The firewall permits the transaction since the address falls outside the protected block. The write operation succeeds, modifying the protected register and bypassing the security mechanism.

## MITRE Reference Data

### Description

The firewall in an on-chip fabric protects the main addressed region, but it does not protect any mirrored memory or memory-mapped-IO (MMIO) regions.

### Extended Description

Few fabrics mirror memory and address ranges, where mirrored regions contain copies of the original data. This redundancy is used to achieve fault tolerance. Whatever protections the fabric firewall implements for the original region should also apply to the mirrored regions. If not, an attacker could bypass existing read/write protections by reading from/writing to the mirrored regions to leak or corrupt the original data.

### Demonstrative Examples

#### Example 1

A memory-controller IP block is connected to the on-chip fabric in a System on Chip (SoC).  The memory controller is configured to divide the memory into four parts: one original and three mirrored regions inside the memory. The upper two bits of the address indicate which region is being addressed. 00 indicates the original region and 01, 10, and 11 are used to address the mirrored regions. All four regions operate in a lock-step manner and are always synchronized. The firewall in the on-chip fabric is programmed to protect the assets in the memory.

The firewall only protects the original range but not the mirrored regions.

The attacker (as an unprivileged user) sends a write transaction to the mirrored region. The mirrored region has an address with the upper two bits set to "10" and the remaining bits of the address pointing to an asset. The firewall does not block this write transaction. Once the write is successful, contents in the protected-memory region are also updated. Thus, the attacker can bypass existing, memory protections.

Firewall should protect mirrored regions.

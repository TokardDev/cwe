# CWE-1332 - Improper Handling of Faults that Lead to Instruction Skips


## Related CVEs

| CVE | Description |
|-----|-------------|
| [CVE-2019-15894](https://www.cve.org/CVERecord?id=CVE-2019-15894) | fault injection attack bypasses the verification mode, potentially allowing arbitrary code execution. |

## CWE Explanation

This weakness arises when a hardware device lacks adequate mechanisms to detect and respond to faults that result in the processor skipping instructions. Environmental anomalies, such as voltage fluctuations or clock glitches, can disrupt the normal instruction fetch and execution cycle, causing the CPU to advance the program counter without executing the current instruction.

When security-critical instructions—such as conditional jumps associated with password verification or firmware authentication—are skipped, the device may fail open or bypass essential security checks. This allows an attacker to alter the intended execution flow, effectively neutralizing software-level protection mechanisms by exploiting the hardware's inability to maintain execution integrity under stress.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | The attacker can bypass authentication to access sensitive data |

## Attack Scenario

An attacker has physical access to a device and is able to manipulate its operating environment, such as the power supply voltage. The attacker targets a specific security check, such as a PIN verification routine, where a conditional branch instruction determines access privileges. By injecting a precise voltage glitch exactly when the processor is fetching or decoding the branch instruction, the attacker causes the CPU to skip the instruction entirely. As a result, the program counter increments to the next instruction as if the security check had passed, granting the attacker unauthorized access to the protected system or data.

## MITRE Reference Data

### Description

The device is missing or incorrectly implements circuitry or sensors that detect and mitigate the skipping of security-critical CPU instructions when they occur.

### Extended Description

The operating conditions of hardware may change in ways that cause unexpected behavior to occur, including the skipping of security-critical CPU instructions. Generally, this can occur due to electrical disturbances or when the device operates outside of its expected conditions.

In practice, application code may contain conditional branches that are security-sensitive (e.g., accepting or rejecting a user-provided password). These conditional branches are typically implemented by a single conditional branch instruction in the program binary which, if skipped, may lead to effectively flipping the branch condition - i.e., causing the wrong security-sensitive branch to be taken. This affects processes such as firmware authentication, password verification, and other security-sensitive decision points.

Attackers can use fault injection techniques to alter the operating conditions of hardware so that security-critical instructions are skipped more frequently or more reliably than they would in a "natural" setting.

### Demonstrative Examples

Example 1

A smart card contains authentication credentials that are used as authorization to enter a building. The credentials are only accessible when a correct PIN is presented to the card.

```

The card emits the credentials when a voltage anomaly is injected into the power line to the device at a particular time after providing an incorrect PIN to the card, causing the internal program to accept the incorrect PIN.
```

There are several ways this weakness could be fixed.

```
(good code)

* add an internal filter or internal power supply in series with the power supply pin on the device

* add additional execution sensing circuits to monitor the execution order for anomalies and abort the action or reset the device under fault conditions
```

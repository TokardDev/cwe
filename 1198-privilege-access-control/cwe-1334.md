# CWE-1334 - Unauthorized Error Injection Can Degrade Hardware Redundancy

## CWE Explanation

This weakness occurs when mechanisms designed to inject errors for testing and validation purposes are not properly secured, allowing unauthorized agents to trigger faults in critical hardware blocks.

High-reliability systems often use redundancy (such as Triple Modular Redundancy or backup cores) to ensure continuous operation in the event of hardware failures. To validate these redundancy features during manufacturing or deployment, designers include "error injection" interfaces that can simulate faults.

If access to these error injection interfaces is not restricted to trusted agents (e.g., via authentication or physical disablement), an attacker can maliciously trigger faults. This can force the system to switch to a backup unit, disable redundancy (leaving the system vulnerable to a real natural fault), or enter a degraded "fail-safe" mode where security controls may be relaxed to prioritize availability.

## Impact Assessment

| Aspect | Impact |
|--------|--------|
| **Confidentiality** | Low impact (primarily affects availability and integrity). |
| **Integrity** | An attacker can force the system into a specific state or degrade the reliability of calculations by disabling redundancy checks. |
| **Availability** | An attacker can repeatedly inject errors to exhaust redundant resources, causing a complete system shutdown or denial of service. |

## Attack Scenario

An automotive safety controller uses two redundant processor cores running in lockstep to detect hardware faults. The system includes a memory-mapped "Fault Injection" register designed for factory testing, which allows software to flip bits in the output of Core A to test if the comparator logic correctly detects the mismatch. This register should be locked out in production mode. However, the locking mechanism is not implemented for the specific address range containing this register. In the next step, an attacker who has compromised the infotainment system (running on the same SoC but a different partition) writes to the Fault Injection register. This artificially creates a mismatch between the cores. The system's safety logic detects the error and, assuming a hardware failure, shuts down the "faulty" Core A and switches the system to "Limp Home" mode. In this degraded mode, certain complex safety checks are disabled to save processing power, leaving the vehicle vulnerable to further attacks or unsafe operation.

## MITRE Reference Data

### Description

An unauthorized agent can inject errors into a redundant block to deprive the system of redundancy or put the system in a degraded operating mode.

### Extended Description

To ensure the performance and functional reliability of certain components, hardware designers can implement hardware blocks for redundancy in the case that others fail. This redundant block can be prevented from performing as intended if the design allows unauthorized agents to inject errors into it. In this way, a path with injected errors may become unavailable to serve as a redundant channel. This may put the system into a degraded mode of operation which could be exploited by a subsequent attack.
